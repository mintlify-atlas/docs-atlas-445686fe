## Overview

The IPS Calendar module tracks regulatory deadlines and scheduled reports for pharmaceutical products. It integrates with the main IPS reporting system to streamline report creation and compliance tracking.

## List Calendar Entries

<CodeGroup>
```http GET /api/v1/ips/calendario
GET /api/v1/ips/calendario?limit=10&offset=0&q=PARACETAMOL
Authorization: Bearer <token>
```

```python Python
response = requests.get(
    "https://api.vigia.com/api/v1/ips/calendario",
    headers={"Authorization": f"Bearer {token}"},
    params={
        "limit": 10,
        "offset": 0,
        "q": "PARACETAMOL"
    }
)
data = response.json()
items = data["items"]
total = data["total"]
```
</CodeGroup>

### Query Parameters

<ParamField query="limit" type="integer" default="10">
  Number of entries to return (1-200)
</ParamField>

<ParamField query="offset" type="integer" default="0">
  Number of entries to skip for pagination
</ParamField>

<ParamField query="q" type="string">
  Search query to filter by product name, active ingredient, or registration number
</ParamField>

### Response

<ResponseField name="items" type="array<CalendarioIPSOut>">
  List of calendar entries
</ResponseField>

<ResponseField name="total" type="integer">
  Total number of matching entries
</ResponseField>

## Create Calendar Entry

<CodeGroup>
```http POST /api/v1/ips/calendario
POST /api/v1/ips/calendario
Authorization: Bearer <token>
Content-Type: application/json

{
  "producto_forma": "PARACETAMOL 500MG - TABLETAS",
  "ifa": "PARACETAMOL",
  "registro_numero": "2021-123456",
  "registro_estado": "Vigente",
  "registro_vencimiento": "2026-03-15",
  "registro_primera_autorizacion": "2021-03-15",
  "ips_fid": "2024-01-01",
  "ips_fcd": "2024-12-31",
  "fecha_lim_elaboracion": "2025-01-31",
  "estado_ips": "Borrador"
}
```

```python Python
response = requests.post(
    "https://api.vigia.com/api/v1/ips/calendario",
    headers={"Authorization": f"Bearer {token}"},
    json={
        "producto_forma": "PARACETAMOL 500MG - TABLETAS",
        "ifa": "PARACETAMOL",
        "registro_numero": "2021-123456",
        "registro_estado": "Vigente",
        "ips_fid": "2024-01-01",
        "ips_fcd": "2024-12-31",
        "estado_ips": "Borrador"
    }
)
entry = response.json()
```
</CodeGroup>

### Request Body (CalendarioIPSCreate)

<ParamField body="producto_forma" type="string" required>
  Product name with pharmaceutical form
</ParamField>

<ParamField body="ifa" type="string" required>
  Active ingredient (IFA)
</ParamField>

<ParamField body="registro_numero" type="string" required>
  Sanitary registration number
</ParamField>

<ParamField body="registro_estado" type="string" default="Vigente">
  Registration status
</ParamField>

<ParamField body="registro_vencimiento" type="date">
  Registration expiration date (YYYY-MM-DD)
</ParamField>

<ParamField body="registro_primera_autorizacion" type="date">
  First authorization date (YYYY-MM-DD)
</ParamField>

<ParamField body="ips_version" type="string" default="1.0">
  IPS version number
</ParamField>

<ParamField body="ips_fid" type="date">
  IPS period start date (FID - Fecha Inicio Datos)
</ParamField>

<ParamField body="ips_fcd" type="date">
  IPS period end date (FCD - Fecha Cierre Datos)
</ParamField>

<ParamField body="ips_anios" type="number">
  IPS period duration in years
</ParamField>

<ParamField body="rs_version" type="string" default="1.0">
  Safety summary (Resumen de Seguridad) version
</ParamField>

<ParamField body="rs_fid" type="date">
  Safety summary period start date
</ParamField>

<ParamField body="rs_fcd" type="date">
  Safety summary period end date
</ParamField>

<ParamField body="rs_anios" type="number">
  Safety summary period duration in years
</ParamField>

<ParamField body="fecha_lim_elaboracion" type="date">
  Report elaboration deadline
</ParamField>

<ParamField body="fecha_lim_revision_aprobacion" type="date">
  Review and approval deadline
</ParamField>

<ParamField body="fecha_sometimiento" type="date">
  Submission date to regulatory authority
</ParamField>

<ParamField body="estado_ips" type="string" default="Borrador">
  IPS status: `Borrador`, `Validado`, `Firmado`, `Sometido`, `Aprobado`, or `Observado`
</ParamField>

<ParamField body="responsable_aprobacion" type="string">
  Person responsible for approval
</ParamField>

### Response

Returns the created calendar entry with HTTP 201 Created.

</CodeGroup>

### Path Parameters

<ParamField path="item_id" type="integer" required>
  ID of the calendar entry to update
</ParamField>

### Request Body

Same fields as `CalendarioIPSCreate`. All fields are optional.

Updates are automatically tracked with before/after change detection:

```json

  }
}
```

## Create Report from Calendar

Create an IPS report directly from a calendar entry.

<CodeGroup>
```http POST /api/v1/ips/from-calendario
POST /api/v1/ips/from-calendario?cal_id=789
Authorization: Bearer <token>
Content-Type: application/json

{
  "fuentes": [
    {
      "title": "Safety study",
      "collection": "PMC",
      "year": 2024
    }
  ],
  "kpis": {
    "literaturas_detected": 50
  },
  "unidades_comercializadas": 50000
}
```

```python Python
response = requests.post(
    "https://api.vigia.com/api/v1/ips/from-calendario",
    headers={"Authorization": f"Bearer {token}"},
    params={"cal_id": 789},
    json={
        "fuentes": [{"title": "Safety study", "collection": "PMC"}],
        "unidades_comercializadas": 50000
    }
)
report = response.json()
```
</CodeGroup>

### Query Parameters

<ParamField query="cal_id" type="integer" required>
  Calendar entry ID to create report from
</ParamField>

<ParamField query="periodo_inicio" type="date">
  Override period start date (defaults to calendar entry dates)
</ParamField>

<ParamField query="periodo_fin" type="date">
  Override period end date (defaults to calendar entry dates)
</ParamField>

### Request Body (Optional)

<ParamField body="fuentes" type="array">
  Initial literature sources to attach
</ParamField>

<ParamField body="kpis" type="object">
  Initial KPI values
</ParamField>

<ParamField body="unidades_comercializadas" type="integer">
  Units commercialized during period
</ParamField>

### Response

Returns the created `IPSReportOut` with associated product automatically created or matched.

## View Report History by Calendar

Get all IPS reports associated with a calendar entry.

<CodeGroup>
```http GET /api/v1/ips/calendario/{cal_id}/reports
GET /api/v1/ips/calendario/789/reports?limit=50&offset=0
Authorization: Bearer <token>
```

```python Python
response = requests.get(
    "https://api.vigia.com/api/v1/ips/calendario/789/reports",
    headers={"Authorization": f"Bearer {token}"},
    params={"limit": 50}
)
reports = response.json()
```
</CodeGroup>

### Path Parameters

<ParamField path="cal_id" type="integer" required>
  Calendar entry ID
</ParamField>

### Query Parameters

<ParamField query="limit" type="integer" default="50">
  Number of reports to return (1-200)
</ParamField>

<ParamField query="offset" type="integer" default="0">
  Number of reports to skip
</ParamField>

### Response

Returns an array of `IPSReportOut` objects, ordered by most recent first.
