## POST /api/v1/auth/login

Authenticate a user and receive a JWT access token.

### Headers

<ParamField header="X-Tenant" type="string" optional>
  Tenant identifier for multi-tenant authentication. If not provided, uses legacy mode.
</ParamField>

<ParamField header="Content-Type" type="string" required>
  Must be `application/x-www-form-urlencoded`
</ParamField>

### Request Body (Form Data)

<ParamField body="username" type="string" required>
  Username or email address. The system accepts both and searches case-insensitively.
</ParamField>

<ParamField body="password" type="string" required>
  User password (minimum 6 characters)
</ParamField>

<ParamField body="grant_type" type="string" optional>
  OAuth2 grant type. Default: `password`
</ParamField>

<ParamField body="scope" type="string" optional>
  OAuth2 scope (not currently used)
</ParamField>

### Response

<ResponseField name="access_token" type="string" required>
  JWT access token valid for 480 minutes (8 hours)
</ResponseField>

<ResponseField name="token_type" type="string" required>
  Token type, always returns `"bearer"`
</ResponseField>

<ResponseField name="user" type="object" required>
  User information object
  
  <Expandable title="user properties">
    <ResponseField name="id" type="integer">
      User ID
    </ResponseField>
    
    <ResponseField name="username" type="string">
      Username
    </ResponseField>
    
    <ResponseField name="email" type="string">
      User email address
    </ResponseField>
    
    <ResponseField name="role" type="string">
      Primary role (admin takes precedence if present)
    </ResponseField>
    
    <ResponseField name="roles" type="array">
      List of all assigned role names
    </ResponseField>
  </Expandable>
</ResponseField>

<RequestExample>
```bash cURL
curl -X POST "https://api.vigia.com/api/v1/auth/login" \
  -H "X-Tenant: acme-pharma" \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "username=juan.perez@example.com" \
  -d "password=SecurePass123"
```

```python Python
import requests

url = "https://api.vigia.com/api/v1/auth/login"
headers = {
    "X-Tenant": "acme-pharma"
}
data = {
    "username": "juan.perez@example.com",
    "password": "SecurePass123"
}

response = requests.post(url, headers=headers, data=data)
token_data = response.json()
```


## GET /api/v1/auth/me

Retrieve detailed information about the currently authenticated user.

### Headers

<ParamField header="Authorization" type="string" required>
  Bearer token obtained from login endpoint
  
  Format: `Bearer <token>`
</ParamField>

<ParamField header="X-Tenant" type="string" optional>
  Tenant identifier (extracted from token if not provided)
</ParamField>

### Response

<ResponseField name="id" type="integer" required>
  User ID
</ResponseField>

<ResponseField name="email" type="string" required>
  User email address
</ResponseField>

<ResponseField name="username" type="string" required>
  Username
</ResponseField>

<ResponseField name="full_name" type="string">
  User's full name
</ResponseField>

<ResponseField name="role" type="string">
  Primary role (admin prioritized)
</ResponseField>

<ResponseField name="roles" type="array" required>
  List of all assigned role names
</ResponseField>

<ResponseField name="permissions" type="object" required>
  Custom permissions object (module-specific permissions)
</ResponseField>

<ResponseField name="is_active" type="boolean" required>
  Whether the user account is active
</ResponseField>

<ResponseField name="empleado_id" type="integer">
  Associated employee ID from HR module
</ResponseField>

<RequestExample>
```bash cURL
curl -X GET "https://api.vigia.com/api/v1/auth/me" \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..." \
  -H "X-Tenant: acme-pharma"
```

```python Python
import requests

url = "https://api.vigia.com/api/v1/auth/me"
headers = {
    "Authorization": f"Bearer {access_token}",
    "X-Tenant": "acme-pharma"
```

```javascript JavaScript
});

```
</RequestExample>

## POST /api/v1/auth/change-password

Change the password for the currently authenticated user.

### Headers

<ParamField header="Authorization" type="string" required>
  Bearer token
</ParamField>

<ParamField header="Content-Type" type="string" required>
  Must be `application/json`
</ParamField>

### Request Body

<ParamField body="current_password" type="string" required>
  Current password for verification
</ParamField>

<ParamField body="new_password" type="string" required>
  New password (minimum 6 characters)
</ParamField>

### Response

<ResponseField name="ok" type="boolean" required>
  Success indicator, always `true` on success
</ResponseField>

<RequestExample>
```bash cURL
curl -X POST "https://api.vigia.com/api/v1/auth/change-password" \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..." \
  -H "Content-Type: application/json" \
  -d '{
    "current_password": "OldPassword123",
    "new_password": "NewSecurePass456"
  }'
```

```python Python
import requests

url = "https://api.vigia.com/api/v1/auth/change-password"
headers = {
    "Authorization": f"Bearer {access_token}",
    "Content-Type": "application/json"
}
data = {
    "current_password": "OldPassword123",
    "new_password": "NewSecurePass456"
}

response = requests.post(url, headers=headers, json=data)
```

```javascript JavaScript
const response = await fetch('https://api.vigia.com/api/v1/auth/change-password', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${accessToken}`,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    current_password: 'OldPassword123',
    new_password: 'NewSecurePass456'
  })
});
```
</RequestExample>

<ResponseExample>
```json Success Response (200)
{
  "ok": true
}
```

```json Wrong Current Password (401)
{
  "detail": "Contraseña actual incorrecta"
}
```

```json Password Too Short (422)
{
  "detail": "Nueva contraseña muy corta (mín 6)"
}
```

```json User Not Found (404)
{
  "detail": "Usuario no encontrado"
}
```
</ResponseExample>

## POST /api/v1/auth/reset-password

Reset a user's password (admin/development endpoint).

<Warning>
  This endpoint is intended for administrative use and development/testing. In production, this should be restricted to admin users only.
</Warning>

### Headers

<ParamField header="X-Tenant" type="string" required>
  Tenant identifier (required for this endpoint)
</ParamField>

<ParamField header="Content-Type" type="string" required>
  Must be `application/json`
</ParamField>

### Request Body

<ParamField body="email" type="string" required>
  Email address of the user whose password should be reset
</ParamField>

<ParamField body="new_password" type="string" required>
  New password to set (minimum 6 characters)
</ParamField>

### Response

<ResponseField name="ok" type="boolean" required>
  Success indicator
</ResponseField>

<ResponseField name="email" type="string" required>
  Email of the user whose password was reset
</ResponseField>

<ResponseField name="tenant" type="string" required>
  Tenant where the operation was performed
</ResponseField>

<RequestExample>
```bash cURL
curl -X POST "https://api.vigia.com/api/v1/auth/reset-password" \
  -H "X-Tenant: acme-pharma" \
  -H "Content-Type: application/json" \
  -d '{
    "email": "user@example.com",
    "new_password": "NewPassword123"
  }'
```

```python Python
import requests

url = "https://api.vigia.com/api/v1/auth/reset-password"
```

## Common Error Codes

All authentication endpoints may return these standard HTTP status codes:

| Status Code | Description |
|-------------|-------------|
| 200 | Success |
| 401 | Unauthorized (invalid credentials or token) |
| 403 | Forbidden (inactive user or no roles) |
| 404 | Not Found (user not found) |
| 422 | Unprocessable Entity (validation error) |
| 500 | Internal Server Error |

## Token Usage Example

After obtaining a token from the login endpoint, use it in all subsequent requests:

```python Complete Authentication Flow
import requests

# 1. Login
login_response = requests.post(
    "https://api.vigia.com/api/v1/auth/login",
    headers={"X-Tenant": "acme-pharma"},
    data={
        "username": "juan.perez@example.com",
        "password": "SecurePass123"
    }
)

token = login_response.json()["access_token"]

# 2. Use token for authenticated requests
headers = {
    "Authorization": f"Bearer {token}",
    "X-Tenant": "acme-pharma"
}

# 3. Get current user info
user_info = requests.get(
    "https://api.vigia.com/api/v1/auth/me",
    headers=headers
).json()

# 4. Make other API calls
icsrs = requests.get(
    "https://api.vigia.com/api/v1/icsrs",
    headers=headers
).json()
```
